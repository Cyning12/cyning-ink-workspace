1. **环境**：Neo4j 部署形态（Aura / Docker / 其他）与连接配置归属（不写具体密钥）。

3. **查询层**：与方案2 对齐的函数语义，内部走 Cypher；复杂分析查询单独列表（待补）。
4. **统一接口（可选）**：环境变量或工厂切换内存实现与 Neo4j 实现；**冲突时读 DB（R3）**，内存路径仅作降级或未连库时。
5. **CI 策略**：默认 job 不依赖 Neo4j；若有集成测试，单独 pipeline 或 nightly（需求方定）。
